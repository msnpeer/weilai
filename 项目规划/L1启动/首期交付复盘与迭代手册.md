# 首期交付复盘与迭代手册

> 首期结束后，团队用这份手册完成从"做完了"到"知道怎么做好"的跃迁。
> 目标：把首期经验转化为可复制的标准化模型。

---

## 一、复盘总览

### 1.1 复盘时间线

| 时间节点 | 动作 | 产出 | 负责人 |
|----------|------|------|--------|
| T+0（结营当天） | 现场快速复盘会（30分钟） | 口头要点记录 | 主持人 |
| T+1~T+3 | 学员数据整理 + 团队回顾会（2小时） | 数据报告 + 初步发现 | 运营+讲师 |
| T+4~T+7 | 深度复盘会（半天） | 复盘报告初稿 | 全团队 |
| T+8~T+14 | 迭代方案制定 | 第二期优化方案 | 老板+运营 |
| T+15~T+21 | 文档更新 + 标准化 | 更新后的全套SOP | 运营 |

### 1.2 复盘四问（核心框架）

每个复盘环节都围绕这四个问题展开：

```
① 目标是什么？ → 对照《首期招生目标与策略方案》
② 实际发生了什么？ → 用数据说话
③ 为什么会有差距？ → 根因分析，不甩锅
④ 下次怎么改？ → 具体可执行的改进动作
```

## 二、结营当天快速复盘（T+0）

### 2.1 现场快速复盘会

结营当天离场后，团队留下来开30分钟快速复盘会。

**会议流程：**

| 时间 | 环节 | 内容 |
|------|------|------|
| 0-5min | 情绪收集 | 每人用一个词形容今天的感受（不讨论） |
| 5-15min | 亮点回顾 | 每人说1个今天最满意的瞬间 |
| 15-25min | 问题收集 | 每人说1个今天最大的问题（不讨论解决方案） |
| 25-30min | 行动约定 | 约定深度复盘会时间 + 每人带1份数据来 |

## 五、根因分析与改进方案

### 5.1 问题归因矩阵

将所有发现的问题按以下框架归因：

| 问题描述 | 影响程度(H/M/L) | 根因类别 | 根因描述 | 改进方案 | 优先级 |
|----------|-----------------|----------|----------|----------|--------|
| | | □课程设计 □工具技术 □讲师能力 □助教配合 □运营流程 □物料准备 □招生策略 □学员管理 | | | P0/P1/P2 |
| | | | | | |
| | | | | | |

### 5.2 根因类别说明

| 类别 | 典型表现 | 改进方向 |
|------|----------|----------|
| **课程设计** | 内容太难/太简单、时间分配不合理、模块衔接不顺 | 调整教案、增减内容、优化节奏 |
| **工具技术** | 工具崩溃、注册失败、网络问题、操作太复杂 | 更换工具、增加备用方案、简化操作指引 |
| **讲师能力** | 讲解不清、节奏失控、互动不足、应变不够 | 培训讲师、提供话术脚本、增加备课时间 |
| **助教配合** | 辅导不及时、标准不统一、站位不合理 | 培训助教、明确分工、制定巡场SOP |
| **运营流程** | 签到混乱、物料缺失、时间延误、沟通不畅 | 优化SOP、增加检查点、明确责任人 |
| **物料准备** | 打印不足、设备故障、物料缺失 | 完善物料清单、增加冗余、提前检查 |
| **招生策略** | 人数不达标、赛道不均衡、渠道失效 | 调整渠道、优化话术、增加预算 |
| **学员管理** | 迟到早退、设备未准备、期望不匹配 | 加强课前沟通、设置准入门槛、调整问卷 |

### 5.3 改进优先级定义

| 优先级 | 定义 | 处理时限 | 示例 |
|--------|------|----------|------|
| **P0** | 阻塞性问题，不解决下期无法开课 | 立即处理 | 工具不可用、讲师缺失、场地问题 |
| **P1** | 严重影响学员体验或收入 | 2周内解决 | 课程时间不够、转化率过低、物料不足 |
| **P2** | 优化型改进，提升质量 | 下期开课前 | 话术优化、分组策略调整、互动环节增加 |

## 十、附录

### 附录A：复盘会主持话术

**开场（2分钟）：**
> 各位辛苦了！今天我们不讲客套话，直接聊干货。目标只有一个：把今天的经验变成第二期更好的表现。每个人的意见都重要，没有对错，只有事实和改进。

**数据回顾（15分钟）：**
> 先看数字。数字不会骗人。大家看看这些数据，有没有跟你的感受不一样的地方？

**亮点回顾（30分钟）：**
> 先说好的。每个模块说一个最满意的点。记住这个感觉，这是我们第二期要保持的。

**问题收集（30分钟）：**
> 现在说问题。注意，我们今天只收集问题，不讨论解决方案。先把问题全部列出来，再统一分析。

**根因分析（30分钟）：**
> 问题列完了，现在逐个看。每个问题问一句"为什么会发生"，找到根本原因，不要停在表面。

**改进方案（30分钟）：**
> 最后一步，每个问题对应一个具体的改进动作。要具体到"谁、什么时候、做什么"。

**收尾（5分钟）：**
> 今天的复盘非常有价值。接下来一周，我会把今天的讨论整理成复盘报告，下周发给大家确认。第二期，我们一起做得更好。
