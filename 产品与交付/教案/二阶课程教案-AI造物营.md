> 内部交付文档 · 讲师/助教专用
> 最后更新：2026-07-02

## 一、课程总览

| 项目 | 内容 |
|------|------|
| 课程名称 | AI 造物营 |
| 时长 | 2 天（每天约 7 小时有效教学，共 14 小时） |
| 定位 | 主力营收款 / 产品创造型 |
| 人数 | 建议 15-25 人/班（比一阶少，因需要更多指导） |
| 师资 | 讲师 1 人 + 助教 3-4 人（1:5-1:6） |
| 设备 | 学员自带笔记本电脑（必须，建站需浏览器操作） |
| 网络 | 必须有稳定 Wi-Fi，建议 100M 以上带宽 |

### 学员标准成果物（离场时每人必须带走）

1. **1 个能访问的网站** — 浏览器可打开、有完整页面结构
2. **1 套 AI 辅助建站 SOP** — 从需求描述到上线的完整流程文档
3. **1 个可演示的专属 Agent** — 能跑通、能讲解逻辑

### 前置条件

- 建议已完成一阶，或具备基础 AI 工具使用经验
- 学员需有明确的产品/作品/展示方向（课前引导确定）
- 必须带笔记本电脑（台式机/平板不适用建站环节）

### Day 1 上午 · 开场（09:00-09:30）

#### 讲师话术脚本

> "各位好，欢迎来到 AI 造物营。一阶的时候，你们学会了用 AI 做内容——海报、视频、文案。今天和明天，我们要做一件更酷的事：**用 AI 做一个属于你自己的真实产品**。"
>
> "不是练习，不是演示，是一个别人可以打开、可以访问、可以分享的东西。"
>
> "在开始之前，我需要每个人确定一个方向——你这两天想做什么？"

#### 引导学员选题

**青少年赛道方向（讲师可推荐）：**
- 个人作品集网站（展示自己的画作、作文、摄影作品）
- 学科学习工具页（数学公式速查、英语单词卡）
- 项目展示页（科技竞赛项目、课外活动记录）
- 兴趣主页（追星/游戏/动漫/运动主题）

**成人赛道方向（讲师可推荐）：**
- 公司展示主页（企业介绍、产品展示）
- 个人品牌页（自由职业者/咨询师/讲师）
- 产品落地页（新产品发布、活动报名）
- 项目提案页（给客户看的方案展示）

#### 选题引导话术

> "不知道做什么？没关系。想想你平时最想给别人看的是什么——你的作品、你的公司、你的项目、你的爱好。把它做成一个网页，就是你这两天的任务。"
>
> "如果实在想不到，可以先看看这几个案例。"（展示 3-5 个模板网站）

#### 阶段检查点

- [ ] 每位学员确定了产品方向
- [ ] 讲师记录每位学员的选题（便于后续个性化指导）

### Day 1 上午 · 模块一：Agent 基础认知与实战（09:30-12:00）

#### 模块目标

让学员理解 Agent 的核心概念，并用扣子 Coze 搭建出第一个可跑通的专属 Agent。

#### 模块一·上：Agent 基础认知（09:30-10:30）

##### 讲师话术脚本

> "什么是 Agent？简单说，就是**一个能帮你自动完成任务的 AI 助手**。"
>
> "你跟豆包聊天，是你问它答。但 Agent 不一样——你给它一个任务，它会自己拆步骤、调工具、出结果。"
>
> "举个例子：你告诉 Agent '帮我整理这周的学习笔记'，它会自动分类、提取重点、生成摘要。这就是 Agent。"

##### 核心知识点（讲师演示）

**1. Agent 三要素**
- **输入**：用户给什么信息（文本/文件/指令）
- **处理**：Agent 怎么拆解任务（流程节点、判断逻辑）
- **输出**：最终交付什么结果（文案/页面/数据）

**2. 扣子 Coze 界面导览**
- 创建新 Bot
- 设置系统提示词（角色定义）
- 添加插件/工具
- 设置工作流节点
- 测试运行

**3. 演示案例**

青少年赛道演示：
> "我做一个'学习助手 Agent'——输入一道数学题，它会给出解题步骤、相关知识点、类似练习题。"

成人赛道演示：
> "我做一个'周报助手 Agent'——输入本周工作要点，它自动生成结构化周报，包含本周成果、下周计划、需要协调的事项。"

##### 学员实操任务

每位学员在扣子 Coze 中：
1. 创建一个新 Bot
2. 设置角色名称和系统提示词
3. 添加 1-2 个基础插件
4. 完成一次测试对话

##### 阶段检查点

- [ ] 学员理解 Agent 的输入-处理-输出逻辑
- [ ] 每位学员创建了 Bot 并完成测试

#### 休息（10:30-10:45）

#### 模块一·下：搭建第一个专属 Agent（10:45-12:00）

##### 讲师话术脚本

> "刚才我们了解了 Agent 是什么。现在，每个人要搭建一个**跟你自己的方向相关的 Agent**。"
>
> "如果你做的是作品集网站，可以搭一个'作品介绍 Agent'——输入作品名称，自动生成作品描述、创作背景、技术说明。"
>
> "如果你做的是公司主页，可以搭一个'公司介绍 Agent'——输入公司业务关键词，自动生成公司简介、服务介绍、团队介绍。"

##### 学员实操任务

**青少年赛道任务：**
- 搭建一个与自己选题相关的 Agent
- 示例：作品介绍 Agent / 学习助手 Agent / 兴趣推荐 Agent
- 要求：至少设置 3 个流程节点，能处理不同输入

**成人赛道任务：**
- 搭建一个与自己业务相关的 Agent
- 示例：公司介绍 Agent / 产品文案 Agent / 客户FAQ Agent
- 要求：至少设置 3 个流程节点，输出结果可用于网站内容

##### 助教行动指南

- 巡场检查每位学员的 Agent 搭建进度
- 帮助卡壳学员理清思路：你的 Agent 要解决什么问题？输入是什么？输出是什么？
- 鼓励学员调试提示词，优化输出质量
- 记录完成较好的学员（下午展示用）

##### 阶段检查点

- [ ] 每位学员跑通了 1 个专属 Agent
- [ ] Agent 能处理至少 2 种不同输入
- [ ] 学员能口头讲解 Agent 的逻辑

### Day 1 下午 · 模块二：自动化任务实战（13:30-16:30）

#### 模块目标

让学员理解自动化思维，并用 OpenClaw 完成一个真实自动化任务。

#### 模块二·上：自动化思维 + OpenClaw 入门（13:30-14:30）

##### 讲师话术脚本

> "上午我们做了 Agent，它能帮你处理一个任务。但如果你有**10 个、100 个类似的任务**呢？一个个手动处理？"
>
> "这就是自动化要解决的问题。自动化的核心不是'会写代码'，而是**把重复的事情交给机器做**。"
>
> "今天我带大家用 OpenClaw 来体验这个过程。不需要会编程，只需要告诉它你要做什么。"

##### 核心知识点（讲师演示）

**1. 什么是自动化**
- 手动做 vs 自动做的区别
- 自动化的典型场景：批量处理、定时执行、信息抓取
- 自动化的核心：定义规则 → 批量执行 → 检查结果

**2. OpenClaw 界面导览**
- 创建新任务
- 定义输入（文件/网址/数据）
- 设置处理规则
- 运行并查看结果

**3. 演示案例**

青少年赛道演示：
> "我用 OpenClaw 批量抓取 10 个科普网站的标题和摘要，自动整理成一份学习资料清单。"

成人赛道演示：
> "我用 OpenClaw 批量抓取竞品网站的产品信息，自动整理成一份竞品分析表。"

##### 学员实操任务

每位学员：
1. 打开 OpenClaw，创建一个新任务
2. 定义一个与自己选题相关的自动化任务
3. 完成一次运行，查看结果

##### 阶段检查点

- [ ] 学员理解"定义规则→批量执行→检查结果"的自动化逻辑
- [ ] 每位学员创建了自动化任务并运行

#### 休息（14:30-14:45）

#### 模块二·下：实战自动化任务（14:45-16:00）

##### 学员实操任务

**青少年赛道任务：**
- 用 OpenClaw 批量收集与自己主题相关的素材（图片链接、文字资料）
- 自动整理成结构化文档
- 输出：一份可用于网站内容的素材库

**成人赛道任务：**
- 用 OpenClaw 批量收集行业/竞品/产品相关信息
- 自动整理成结构化表格或文档
- 输出：一份可用于网站内容的素材库

##### 助教行动指南

- 帮助学员明确"要自动化什么"——很多学员会卡在不知道做什么
- 引导学员从自己的网站选题出发，找一个可自动化的子任务
- 记录完成较好的自动化任务（结营展示用）

##### 阶段检查点

- [ ] 每位学员完成 1 个自动化任务
- [ ] 任务结果可用于 Day 2 建站
- [ ] 学员能解释自己的自动化逻辑

### Day 2 上午 · 回顾 + 模块三：AI 辅助建站（09:00-12:00）

#### Day 2 回顾（09:00-09:30）

##### 讲师话术脚本

> "昨天我们做了两件事：Agent 和自动化。今天我们要把它们变成一个**真实的产品**。"
>
> "今天的流程很简单：先设计你要做什么页面，再用 AI 生成代码，最后调好内容上线。整个过程不需要你会写代码——AI 来写，你来指挥。"

#### 模块三·上：页面需求描述 + 结构设计（09:30-10:30）

##### 讲师话术脚本

> "建站的第一步不是打开电脑写代码，而是**想清楚你要做什么**。"
>
> "你需要回答三个问题：
> 1. 这个网站给谁看？
> 2. 他们打开网站第一眼应该看到什么？
> 3. 你希望他们看完之后做什么？"

##### 核心知识点

**1. 页面结构基础**
- 首页（Hero 区域：标题 + 副标题 + CTA）
- 内容区（核心信息、产品/作品展示）
- 关于区（自我介绍/公司介绍）
- 联系区（联系方式/社交媒体）

**2. 需求描述模板**

讲师提供模板：
```
我要做一个 [类型] 网站，给 [目标用户] 看。
网站名称是 [名称]。
首页要展示 [核心内容1]、[核心内容2]、[核心内容3]。
风格偏 [简约/活泼/专业/科技感]。
配色方案：[主色调] + [辅助色]。
参考网站：[链接]（可选）
```

##### 学员实操任务

每位学员填写需求描述文档：
1. 确定网站类型和目标用户
2. 列出首页要展示的核心内容（至少 3 项）
3. 确定风格和配色
4. 画出简单的页面结构草图（纸笔即可）

**青少年赛道引导：**
- 作品集网站：首页放最好的 3 件作品 + 个人介绍
- 学习工具页：首页放核心功能入口 + 使用说明

**成人赛道引导：**
- 公司主页：首页放公司简介 + 核心产品/服务 + 联系方式
- 个人品牌页：首页放个人标签 + 代表作品 + 服务介绍

##### 阶段检查点

- [ ] 每位学员完成了需求描述文档
- [ ] 页面结构草图清晰

#### 休息（10:30-10:45）

#### 模块三·下：AI 生成页面代码 + 初步调试（10:45-12:00）

##### 讲师话术脚本

> "需求写好了，现在我们让 AI 来写代码。"
>
> "我会用 Claude Code 演示一遍——把需求描述发给它，它会自动生成完整的网页代码。你不需要看懂代码，只需要看效果，然后告诉它哪里要改。"

##### 讲师演示流程

**Step 1：需求转代码**
- 打开 Claude Code
- 粘贴需求描述
- AI 生成 HTML/CSS/JS 代码
- 复制代码，在浏览器中打开预览

**Step 2：初步调试**
- 发现问题 → 描述问题 → AI 修改代码 → 重新预览
- 常见问题：布局错乱、图片不显示、文字太多/太少

**Step 3：对照工具**
- 用 Codex 生成同样需求的页面，对比效果
- 让学员理解不同工具的风格差异

##### 学员实操任务

每位学员：
1. 把需求描述发给 Claude Code
2. 获取生成的代码
3. 在浏览器中预览效果
4. 至少进行 1 次修改（告诉 AI 哪里不对，让它改）
5. 保存当前版本

##### 助教行动指南

- 帮助不知道怎么描述需求的学员——用提问引导
- 帮助代码预览出问题的学员——检查是否复制完整
- 鼓励学员多尝试："AI 生成的第一版不完美很正常，关键是你知道怎么让它改"

##### 阶段检查点

- [ ] 每位学员有了网站第一版页面
- [ ] 页面在浏览器中可预览
- [ ] 学员学会了"描述问题→AI修改→重新预览"的调试循环

### Day 2 下午 · 模块四：产品打磨与上线（13:30-17:00）

#### 模块四·上：页面优化 + 内容填充（13:30-14:30）

##### 讲师话术脚本

> "上午我们有了第一版页面。下午我们要做两件事：**让它变好看，让它有内容**。"
>
> "变好看——调整布局、配色、字体。有内容——把你的真实信息放进去。"

##### 学员实操任务

**页面优化（前 30 分钟）：**
- 调整布局（让 AI 重新排版）
- 优化配色（告诉 AI 你想要的风格）
- 添加图片（用即梦 AI 生成或从素材库选取）
- 调整字体和间距

**内容填充（后 30 分钟）：**
- 用昨天搭建的 Agent 生成网站文案

## 七、工具使用详细指引

### 扣子 Coze（Agent 搭建）

**创建 Bot 流程：**
1. 登录 coze.cn → 创建 Bot
2. 设置名称和描述
3. 编写系统提示词（角色定义 + 任务说明 + 输出格式）
4. 添加插件（搜索、图片生成等）
5. 测试对话 → 优化提示词 → 再测试

**提示词模板：**
```
你是一个 [角色名称]，专门帮助 [目标用户] 完成 [具体任务]。

当用户输入 [输入类型] 时，你需要：
1. [步骤1]
2. [步骤2]
3. [步骤3]

输出格式要求：
- [格式要求1]
- [格式要求2]
```

### Claude Code（AI 建站）

**使用流程：**
1. 打开 Claude Code
2. 粘贴需求描述
3. AI 生成 HTML 代码
4. 复制代码，保存为 index.html
5. 在浏览器中打开预览
6. 描述问题 → AI 修改 → 重新预览

**常见修改指令：**
- "标题改成 [新标题]"
- "背景色改成 [颜色]"
- "在 [位置] 加一张图片"
- "把这段文字缩短到 [字数]"
- "让页面在手机上也能正常显示"

### OpenClaw（自动化任务）

**创建任务流程：**
1. 登录 OpenClaw
2. 创建新任务
3. 定义输入（网址/文件/数据）
4. 设置处理规则
5. 运行任务
6. 查看并导出结果

## 八、应急处理手册

### 网络问题

| 情况 | 处理方案 |
|------|----------|
| Wi-Fi 断线 | 立即启用备用 4G 热点 |
| 工具网站打不开 | 切换备用工具（如 Claude Code 不可用，用 Codex） |
| 速度太慢 | 关闭其他标签页，减少同时操作人数 |

### 工具问题

| 情况 | 处理方案 |
|------|----------|
| 扣子 Coze 限额 | 使用备用账号 |
| Claude Code 生成失败 | 换一种需求描述方式，或用 Codex |
| GitHub Pages 部署失败 | 切换到 Vercel/Netlify |
| 某工具临时不可用 | 使用备选工具完成同类型任务 |

### 学员问题

| 情况 | 处理方案 |
|------|----------|
| 学员基础差异大 | 助教重点帮助慢的学员，快的学员做进阶任务 |
| 学员不知道做什么 | 用选题引导模板，给 3 个推荐方向 |
| 学员中途放弃 | 鼓励 + 降低难度 + 助教一对一帮助 |
| 学员完成太快 | 引导做进阶任务：美化页面、优化 Agent、添加更多内容 |

### 严重故障

| 情况 | 处理方案 |
|------|----------|
| 全部工具不可用 | 切换到理论+案例讲解模式，记录问题，课后补实操 |
| 讲师设备故障 | 使用助教设备继续，提前准备备用设备 |
| 学员电脑故障 | 使用备用电脑或与邻座学员共用 |
