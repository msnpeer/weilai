# Agent 协作规范

> 本文档是 AI Agent 操作本 wiki 的行为准则。每个 Agent 在读写 wiki 前必须先读此文件。

## 核心规则

### 1. 读写前必须同步

```
操作前：git pull origin main
操作后：git add -A && git commit -m "描述做了什么" && git push origin main
```

如果 pull 时有冲突，不要自行合并——告知你的用户，由人来决策。

### 2. 先查后写

接到任务后，先检索 wiki 中是否有相关上下文：

- `wiki/decisions/` — 有没有已经做过的相关决策？
- `wiki/specs/` — 有没有相关的参数/规格约束？
- `wiki/meetings/` — 有没有讨论过类似话题？

不要从零开始猜测，不要在不知道已有决策的情况下做重复决定。

### 3. 写什么、不写什么

**应该写入 wiki 的内容**：
- 项目决策和结论
- 产品参数变更（定价、分成比例等）
- 重要的讨论结论
- 新增的联系人或资源
- 发现的信息不一致或错误

**不应该写入 wiki 的内容**：
- 临时性的操作记录
- 个人工作日志
- 敏感信息（密码、密钥等）
- 推测性或未确认的内容

### 4. 文件命名规范

- 文件名用中文，不用编号前缀
- 决策文件格式：`YYYY-MM-DD_决策主题.md`
- 会议文件格式：`YYYY-MM-DD_会议主题.md`

### 5. 发现信息冲突时

如果你发现 wiki 中两处信息矛盾，或者 wiki 中的信息与你已知的信息不一致：

1. 在对应文件中标注 `<!-- CONFLICT: 描述矛盾 -->`
2. 不要自行选择哪一方为准
3. 告知用户，由人来做裁决
