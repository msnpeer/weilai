# 2026-07-07 Wiki 技术选型

## 背景

团队多人、多设备、多 Agent 协作，信息碎片化严重。需要建立共享知识中枢。

## 决策

**选择 GitHub 私有仓库 + Markdown + VitePress 方案**，作为团队 Wiki 基础设施。

## 理由

1. Markdown 是人和 AI Agent 的最大公约数格式
2. Git 提供版本历史、冲突解决、变更追溯
3. VitePress 自动渲染为网页，手机/浏览器可访问
4. 零额外平台依赖，未来可随时迁移
5. Agent 原生支持本地文件读写

## 备选方案

- 路径 A（Markdown 文件 + 云盘同步）：已否决，同步粒度不够
- 路径 C（Notion / 飞书知识库）：已否决，Agent API 读写不如文件直接

## 实施

- 仓库：github.com/msnpeer/weilai（私有）
- 结构：wiki/ 目录 + .vitepress/ 配置
- 部署：GitHub Actions → GitHub Pages
